# AGENT SPECIFICATION

## 🎯 Цель (Goal)
Каждые 5 минут проверять доступность API:
https://jsonplaceholder.typicode.com/posts/1  
и логировать результат проверки.

---

## 📊 Метрики успеха
API считается доступным, если:
- Статус-код = 200
- Время ответа < 500ms
- В ответе есть поле "id"

---

## ⚙️ Действия при событиях

### ✅ Успех:
- Записать в лог:
  - время
  - статус
  - время ответа

### ❌ Неудача (status != 200):
- Записать ошибку в лог

### 🚨 Критическая ошибка:
(таймаут / сеть)
- Вывести в консоль:
ALERT!

---

## 🧠 Состояние (Memory)
Агент хранит:
- количество подряд ошибок (consecutiveFailures)
- время последнего успешного запроса

---

## 🏗 Архитектура агента

1. Scheduler (node-cron)
2. Probe (HTTP запрос)
3. Analyzer (анализ результата)
4. Logger (запись логов)
5. Dispatcher (решение действий)

---

## 🧰 Используемые инструменты

- axios — HTTP-запросы (есть таймаут)
- node-cron — планировщик
- winston — логирование

Почему:
- axios удобнее fetch для Node.js
- cron даёт гибкое расписание
- winston — промышленный стандарт логов
